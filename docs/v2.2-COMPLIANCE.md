## C9 — CDN & Edge

### Acquis

- ✅ Vercel Edge Network actif par défaut
- ✅ Headers `Cache-Control: no-cache, no-store, must-revalidate` sur `/sw.js`
- ✅ `images.formats: ['image/avif', 'image/webp']` actif

### À améliorer

- 🟡 Pas de stratégie cache-control par type de route documentée
- 🟡 Cloudflare absent — DNS direct OVH → Vercel (acceptable, mais perte WAF + R2)
- 🟡 Aucune route en `runtime = 'edge'` explicite — toutes en Node.js

### Recommandation

Ajouter dans `next.config.ts` une règle cache-control différenciée :

```ts
async headers() {
  return [
    { source: '/(.*)', headers: [/* security headers existants */] },
    { source: '/sw.js', headers: [/* existant */] },
    // NOUVEAU
    {
      source: '/_next/static/(.*)',
      headers: [{ key: 'Cache-Control', value: 'public, max-age=31536000, immutable' }],
    },
    {
      source: '/api/(.*)',
      headers: [{ key: 'Cache-Control', value: 'private, no-store' }],
    },
  ]
}
```

## C10 — Networking & DNS

### Acquis

- ✅ HTTPS + HSTS preload (`max-age=31536000; includeSubDomains; preload`)
- ✅ DNS records OVH : A apex + DKIM/SPF/DMARC validés
- ✅ Rate limiting custom in-memory (`src/proxy.ts` + `src/lib/rate-limit.ts`)
- ✅ CSP stricte (Session 16 PR-5 : `unsafe-eval` retiré)

### À améliorer

- 🟡 Rate limiting **in-memory** = pas de partage entre instances Vercel (chaque cold start réinitialise les compteurs)
  - Recommandation : migrer vers `@upstash/ratelimit` + Upstash Redis (existant pas encore installé) — coût ~$0/mois en hobby
- 🟡 CORS pas explicitement configuré (Vercel autorise par défaut, à durcir si APIs publiques exposées à des domaines tiers)
- 🟡 TLS 1.0/1.1 désactivés ? À vérifier via SSL Labs : https://www.ssllabs.com/ssltest/analyze.html?d=internlog.app

### Vérifications à exécuter

```bash
# DNS records
dig internlog.app +short
dig MX internlog.app +short
dig TXT internlog.app +short | grep "v=spf1"
dig TXT _dmarc.internlog.app +short

# TLS + headers
curl -sI https://internlog.app | grep -i "strict-transport-security\|content-security-policy"

# SSL Labs (manuel, web UI)
```
